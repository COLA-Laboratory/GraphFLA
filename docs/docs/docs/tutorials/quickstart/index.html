    Quickstart

    This section gives a brief overview of the GraphFLA API to analyze a simple fitness landscape.

    Data Loading

    We start with a TrpB landscape from Johnston et al. (2024) ("A combinatorially complete epistatic fitness landscape in an enzyme active site").

    We first load the data and display some random entries in it: 

    import pandas as pd

data = pd.read_csv("data/Johnston2024_TrpB3A.csv")
data.sample(6)

    The dataset contains all \(20^3 = 8,000\) possible combinations of amino acids at 3 sites in Tm9D8 and their associated fitness (in terms of catalytic activity). Most ML-ready biological dataset would have the same information (e.g., the benchmark tasks in ProteinGym), though the sequences can be represented in different forms.

    Date Preparation

    To prepare the data for landscape analysis in GraphFLA is exactly the same as that for training an ML model. All we need is to split the dataset into:

    • X: The genotypes. It can either be a list of sequences or a matrix in pd.DataFrame or np.ndarray.
    • y: The fitness values. It can either be a list of fitness values or a pd.Series or np.ndarray.

    Let's do this:

    X = data["sequences"]
y = data["fitness]

    As we mentioned, X can also be a matrix. In this case, each column indicates the amino acid presents at that position.

    # this also works as the above one for GraphFLA!
X = data[["pos1", "pos2", "pos3",]]

    Landscape Construction

    Now we can create our landscape! This is much like training an ML model: we first initialize a ProteinLandscape object (since we are dealing wth amino acids sequences), and then build it using our prepared data.

    from graphfla.Landscape import ProteinLandscape

landscape = ProteinLandscape(maximize=True)
landscape.build_from_data(X, f, verbose=True)

    The maximize parameter tells GraphFLA whether the fitness value is to be maximized (higher the better) or minimized (lower the better). GraphFLA will then run a landscape constrcution pipeline (as you can see in the following output).

    Landscape Analysis

    With our constructed landscape, we can apply a rich collection of fitness landscape analysis methods to infer its topography and characteristics, including:

    • ruggedness
    • epistasis
    • navigability
    • neutrality
    • fitness distribution

    All analysis methods in GraphFLA have an unified API, and we demonstrate the usage via interrogating the prevalence of different types of epistasis interactions with classify_epistasis().

    from graphfla.analysis import classify_epistasis

classify_epistasis(landscape, approximate=True)
    Output 
    {
    'magnitude epistasis': 0.38521046187197655,
    'sign epistasis': 0.36014372026736785,
    'reciprocal sign epistasis': 0.2546458178606556,
    'positive epistasis': 0.8106778952781974,
    'negative epistasis': 0.18932210472180255
}

    Applying this, we find that this TrpB landscape has abundant non-magnitude epistasis, which gives rise to the 62 local peaks as reported in landscape.describe() before. Also, most epistasis are positive, with only 18.9% negative epistasis.

    Other analysis methods can be applied in a similar manner, and users can select according to their own analysis goals.
