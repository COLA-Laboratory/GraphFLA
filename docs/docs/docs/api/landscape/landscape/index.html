    graphfla.Landscape

    class graphfla.landscape.Landscape(type='default', maximize=True)

    Class implementing the fitness landscape object.

    This class provides a foundational structure for fitness landscapes, conceptualized as a mapping from genotype (configuration) space to fitness values. It represents the landscape as a directed graph (Smith 1970, Aguilar-Rodríguez 2017) where nodes are genotypes and edges connect mutational neighbors, which always point towards fitter variants. Fitness values (and other potential attributes) are stored as node attributes. Fitness differences between neighboring genotypes are stored as edge weights.

    The landscape can be either constructed from raw data (using build_from_data()) or leveraging an existing graph (using build_from_graph()).

    Parameters

    • type : {'default', 'boolean', 'dna', 'rna', 'protein', 'sequence'}, default='default'
      The type of landscape to create.

      • With 'default' it implements a general landscape object that can actually handle all types of data (i.e., ordinal, categorical or boolean). But data_types in build_from_data must be explicitly specified since graphfla does not know what types of data it is handling.
      • With specialized types, i.e., 'boolean', 'dna', 'rna', or 'protein', no explicit data type declariation is needed as they are predefined in graphfla. This would also allow improved performance and more flexible input data formats (see build_from_data). Using these types with Landscape class is equivalent to initializing a specialized landscape class, e.g., DNALandscape.
      • With "sequence", this essentially does the same as SequenceLandscape, which can take performance advantages like with the pre-defined special types above while allow customized search space to handle other problems (e.g., customized chemical space).

    • maximize : bool, default=True
      Determines the optimization direction. If True, the landscape seeks higher fitness values (peaks are optima). If False, it seeks lower fitness values (valleys are optima). This affects edge direction in the graph and identification of optima.

    Attributes

    • graph : igraph.Graph
      The directed igraph.Graph representing the fitness landscape. Nodes represent configurations (genotypes) and edges connect neighboring configurations, typically pointing from lower to higher fitness if maximize is True. Each node usually has a 'fitness' attribute. Populated after calling build_from_data or build_from_graph. Fitness difference between neighboring nodes is stored in the 'delta_fit' edge attribute. Running some analyses, e.g., determine_basin_of_attraction() can result in further node attributes, e.g., "basin_size_greedy".

    • configs : pandas.Series
      A pandas Series mapping node indices (int) to their corresponding configuration representation (often a tuple). This represents the genotypes in the landscape. Populated after calling build_from_data.

    • config_dict : dict
      A dictionary describing the encoding scheme for configuration variables. Keys are typically integer indices of variables, and values are dictionaries specifying properties like 'type' (e.g., 'boolean', 'categorical') and 'max' (maximum encoded value). Populated after calling build_from_data.

    • data_types : dict
      A dictionary specifying the data type for each variable in the configuration space (e.g., {'var_0': 'boolean', 'var_1': 'categorical'}). Validated and stored during build_from_data. Required for certain distance calculations and type-specific processing.

    • n_configs : int
      The total number of configurations (nodes) in the landscape graph. Populated after calling build_from_data or build_from_graph.

    • n_vars : int
      The number of variables (dimensions) defining a configuration in the genotype space. Populated after calling build_from_data or inferred by build_from_graph.

    • shape : tuple
      The shape (n_configs, n_edges) of the landscape graph as a tuple (number_of_nodes, number_of_edges).

    • n_edges : int
      The total number of directed edges (connections) in the landscape graph. Populated after calling build_from_data or build_from_graph.

    • n_lo : int
      The number of local optima (peaks or valleys depending on maximize) in the landscape. Local optima are nodes without fitter neighbors. Populated after graph analysis within build_from_data or build_from_graph.

    • lo : list[int]
      A sorted list of the node indices corresponding to local optima. Populated after graph analysis.

    • go : int
      The node index of the global optimum (the configuration with the highest or lowest fitness). Populated after graph analysis.

    • go : dict
      A dictionary containing the attributes (including 'fitness') of the global optimum node. Populated after graph analysis.

    • lon : igraph.Graph
      The Local Optima Network (LON) (Verel 2010) graph, if calculated via get_lon. Nodes in the LON are local optima from the main landscape graph, and edges represent accessibility between their basins

    • has_lon : bool
      Flag indicating whether the LON has been calculated and stored in the lon attribute. Updated by get_lon.

    • maximize : bool, default=True
      Indicates whether the objective is to maximize (True) or minimize (False) the fitness values. Set during initialization or inferred from graph.

    • epsilon : float or None, default=None
      Tolerance for fitness comparison. Fitness difference less than epsilon will be considered neutral. Set during build_from_data.

    • verbose : bool
      The verbosity level set during initialization or construction. Controls printed output during operations like building or analysis.

    • shape : tuple
      Returns the shape (n_configs, n_edges) of the landscape graph. Accessible via the shape property.

    • _is_built : bool
      Internal flag indicating if the landscape has been populated via build_from_data or build_from_graph.

    • Andrei Papkou et al., "A rugged yet easily navigable fitness landscape", Science (2023).

    Examples

    >>> from graphfla.landscape import Landscape
>>> import pandas as pd

# Example Data (Boolean landscape)
>>> X = pd.DataFrame({
        'var_0': [0, 0, 1, 1], 
        'var_1': [0, 1, 0, 1]
    })
>>> f = pd.Series([
        1.0, 2.0, 3.0, 2.5
    ])
>>> data_types_dict = {'var_0': 'boolean', 'var_1': 'boolean'}

>>> # Initialize and build the landscape
>>> landscape = Landscape(type="boolean", maximize=True)
>>> landscape.build_from_data(
        X, f, 
        data_types=data_types_dict, 
        calculate_basins=True,
        calculate_distance=True
        calculate_accessibility=False,
        calculate_neighbor_fit=False,
        verbose=False
    )

>>> print(f"Number of configurations: {landscape.n_configs}")
Number of configurations: 4
>>> print(f"Global optimum fitness: {landscape.go['fitness']}")
Global optimum fitness: 3.0

>>> # Get data as DataFrame
>>> df = landscape.get_data()
>>> print(df.head())
   var_0  var_1  fitness  is_lo  basin_index  dist_go
0      0      0      1.0  False            2        2
1      0      1      2.0  False            2        1
2      1      0      3.0   True            2        1
3      1      1      2.5  False            2        1

    Methods

    .build_from_data(X, f, attributes=None, data_types=None, epsilon=None, calculate_basins=True, calculate_accessibility=True, calculate_distance=True, calculate_neighbor_fit=True, impute_model=None, filter=None, giant_component=True, verbose=None) -> None

    Construct the landscape graph and properties from configuration data. This method populates the core attributes of the Landscape instance and enable further analysis.

    This method takes genotype-phenotype data (configurations X and their corresponding fitness values f) and builds the underlying graph structure of the fitness landscape. Nodes represent configurations, and edges connect neighbors that differ at only one bit. It then calculates various basic graph/landscape properties like node degrees, local optima, the global optimum, and various advanced features if specified (e.g., with calculate_distance=True the distance of each configuration to the global optimum will be calculated).

    Warning

    Either build_from_data or build_from_graph must be called before any analysis or visualization methods can be applied to a Landscape object.

    Note

    After landscape construction, indices of X and f will be reset to range(len(X)), and f will be named as "fitness" regardless of what they are in the inputs.

    Parameters

    • X : pandas.DataFrame, list[list] or numpy.ndarray
      The configuration data, where each row represents a genotype or configuration, and columns represent variables or sites.

    • f : pandas.Series, list, or numpy.ndarray
      The fitness values corresponding to each configuration in X. Must have the same length as X.

    • attributes : pandas.Series, list, pandas.DataFrame, list[list], or numpy.ndarray
      Additional attribute(s) (e.g., a boolean flag "is_functional") that provide(s) additional information for each configuration.

    • data_types : dict[str, str], optional
      A dictionary specifying the type of each variable (column in X if DataFrame, or inferred column index if ndarray). Keys must match column names/indices, and values must be one of 'boolean', 'categorical', or 'ordinal'. Required if type='default' is used when initializing the landscape, otherwise inferred or generated automatically.

    • epsilon : float, optional
      Noise tolerance \(\epsilon\) used for fitness comparisons. Affects whether small fitness differences are considered neutral or directional, for example, \(\Delta f = f(\mathbf{x}) - f(\mathbf{x}^\prime)\) will be considered 0 (i.e., "neutral") if \(\Delta f < \epsilon\). Typically, a larger \(\epsilon\) would smoothen a landscape that could have been rugged (potentially due to noise in fitness measure) (Aguilar-Rodríguez 2017). In literature, the setting of \(\epsilon\) can be grounded on the variance in fitness measures across replications (Aguilar-Rodríguez 2017).

    • calculate_basins : bool, default=True
      If True, calculates the basins of attraction for each local optimum by calling determine_basin_of_attration(). Populates the node attribute basin_index specifying the basin of the local optimum in which each configuration is located; and size_basin_greedy and radius_basin_greedy for local optima configurations .

    • calculate_accessibility : bool, default=True
      If True, calculates the number of configurations from which the global optimum can be reached via monotonic, fitness-increasing paths by calling determine_accessible_paths(). Populates the size_basin_accessible node attribute. This basin size is the type analyzed in (Papkou 2023). Note that this method can be computationally intensive for large landscapes (e.g., >300,000 configurations) with many local optima.

    • calculate_distance : bool, default=True
      If True, calculates the distance (e.g., hamming distance by default for type='DNA') between each node and the global optimum by calling determine_dist_to_go(). Populates the dist_go node attribute.

    • calculate_neighbor_fit : bool, default=True
      If True, calculates the mean fitness of neighbors for each node (mean_neighbor_fit attribute) and the difference in this mean between connected nodes (delta_mean_neighbor_fit edge attribute) by calling determine_neighbor_fitness(). Derived from Wagner 2023.

    • impute_model : object, optional
      A custom model object with fit and predict methods used for imputing missing fitness values if any. If None a RandomForestRegressor from scikit-learn is used by default when needed.

    • filter : LandscapeFilter, optional
      A filter object (inheriting from graphfla.filters.LandscapeFilter) to apply to the constructed landscape to filter certain nodes and edges based on configurations, fitness, or attributes (e.g., edges where both nodes are labeled as non-functional, see Papkou 2023).

    • giant_component : bool, default=True
      Whether to keep only the largest connected component of the landscape after construction (and filtering if filter is provided).

    • verbose : bool, default=True
      If provided, overrides the instance's verbosity setting for this method call. Controls printed output during the build process.

    Returns

    • None
      This method modifies the Landscape instance in place, populating its attributes (graph, n_configs, lo, etc.). It does not return a new object.

    Warning

    calculate_accessibility can be quite computationally expensive for large landscapes (e.g., >300,000 configurations) with many local optima.

    .build_from_graph(filepath, verbose=True) -> None Construct a landscape from a saved graph file.

    This class method creates a new landscape instance by loading a previously saved graph (in GraphML format, preferably by Landscape.to_graph()). This is significantly faster than building from scratch. It attempts to load essential landscape properties (maximize, epsilon, etc.) stored as graph attributes.

    Parameters

    • filepath : str
      Path to the saved graph file (typically ending in .graphml).
    • verbose : bool, default=True
      Controls verbosity of output during loading and analysis. If True, prints status messages and warnings during the loading process.

    Returns

    • Landscape
      A new instance of the Landscape (or potentially a subclass if the landscape_class attribute is stored in the graph) populated with the loaded graph and inferred properties.

    .to_graph(filepath) -> None Save the landscape graph and essential attributes to a file.

    This method serializes the landscape's graph structure and relevant attributes (maximize, epsilon, data_types, class name) to a GraphML file. This file can later be loaded efficiently using the build_from_graph() class method.

    Parameters

    • filepath : str
      The path where the graph file will be saved. If the file doesn't end with .graphml, this extension will be added automatically.

    Returns

    • None
      This method saves the graph to a file and does not return any value.

    Notes

    • The GraphML format preserves the graph structure and all vertex/edge attributes.
    • Essential landscape attributes are stored as graph-level attributes for reconstruction.
    • Configurations and related dictionaries are converted to string representations for storage.

    .get_data(lo_only=False) -> pandas.DataFrame

    Extracts landscape data as a pandas DataFrame.

    Returns a DataFrame where rows correspond to configurations (nodes) and columns correspond to their attributes (e.g., fitness, degree, basin information, original features).

    Parameters

    • lo_only : bool, default=False
      • If True, returns data only for the configurations identified as local optima (self.lo).
        • If the Local Optima Network (LON) has been computed (self.has_lon is True), relevant attributes from the LON graph (self.lon) are included, prefixed with lon_ (e.g., lon_in_degree, lon_escape_difficulty).
        • If LON is not computed, it returns data from the main graph filtered for local optima nodes.
      • If False (default), returns data for all configurations in the main landscape graph (self.graph), excluding local optima-related attributes (e.g., basin_size_greedy).

    Returns

    • pandas.DataFrame
      A DataFrame containing the attributes of the requested landscape nodes (all or local optima). The index corresponds to the node indices in the original graph. Original feature columns (e.g., 'var_0', 'pos_1') are included if configs was derived from a DataFrame.

    .get_lon(mlon=True, min_edge_freq=3, trim=None, **kwargs) -> igraph.Graph

    Constructs and returns the Local Optima Network (LON) (Verel 2010).

    The LON is a coarse-grained representation of the fitness landscape where nodes are the local optima of the original landscape graph (self.graph), and edges represent the possibility of transitions between their basins of attraction. Edges are typically weighted by the frequency of transitions (escape frequency) between the basins of the connected local optima. This method utilizes the underlying graphfla.landscape.lon.get_lon function for the construction.

    Parameters

    • mlon : bool, default=True
      Whether to construct a Monotonic-LON (M-LON). If True, only edges representing transitions to fitter (or equally fit, depending on the exact implementation of improving edges) local optima are kept in the resulting network. If False, all transitions meeting the min_edge_freq criterion are included.

    • min_edge_freq : int, default=3
      The minimum escape frequency required to create an edge between two local optima. The escape frequency from basin A to basin B is the number of 2-mutant neighbors of local optimum A that fall into the basin of local optimum B. An edge is only created if this frequency is greater than min_edge_freq.

    • trim : int, optional, default=None
      If specified, the LON is trimmed to keep only the top trim outgoing edges from each local optimum node, based on the highest transition frequency (edge weight). If None, all edges meeting the min_edge_freq criterion are kept.

    • *kwargs : Keyword arguments*
      Additional keyword arguments are passed directly to the underlying graphfla.landscape.lon.get_lon function. Consult its specific documentation if needed, though the main configurable parameters are listed above.

    Returns

    • igraph.Graph
      The constructed Local Optima Network graph. This graph is also stored in the self.lon attribute of the Landscape instance, and the self.has_lon flag is set to True. Nodes in the LON graph have attributes copied from the corresponding local optima in the main landscape graph, plus additional LON-specific attributes like escape_difficulty, improve_rate, and accessibility. Edges have a weight attribute representing the transition frequency.

    .describe() -> None

    Prints a summary description of the landscape properties.

    Provides a quick overview of the landscape's current state, including its size (variables, configurations, connections), the number of identified optima, the optimization direction, and the status of optional analysis calculations (basins, paths, LON).

    .determine_local_optima() -> None

    Identifies local optima nodes in the landscape graph. It is always called by build_from_data().

    Local optima are defined as nodes with no outgoing edges to fitter neighbors. In the context of the directed graph (where edges point towards fitter configurations if maximize=True, or less fit if maximize=False), these are nodes with an out-degree of 0. This method calculates the out-degree for all nodes and identifies those where it is zero. It populates the following attributes:

    • Landscape.graph.vs['is_lo']: A boolean flag indicating if each node is a local optimum.
    • Landscape.n_lo: The total count of local optima found.
    • Landscape.lo: A sorted list of the integer indices of the local optima nodes.

    .determine_global_optimum() -> None

    Identifies the global optimum node in the landscape graph. It is always called by build_from_data(). The global optimum is the node with the highest fitness value if maximize=True, or the lowest fitness value if maximize=False. It populates the following attribute:

    • go: The integer index of the global optimum node.

    .determine_basin_of_attraction() -> None

    Calculates the basin of attraction for each node a greedy hill-climbing (a.k.a greedy adaptive walk in biology context, or "best-improvement" hill-climbing in combinatorial optimization, Whitley 2013) algorithm. It is automatically called with build_from_data(calculate_basins=True).

    For each node in the landscape graph, this method simulates a greedy hill-climbing. Starting from the node, it repeatedly moves to the neighbor with the largest fitness improvement (or smallest fitness decrease if minimizing) until a local optimum (a node with no improving neighbors) is reached. The starting node is then assigned to the basin of the reached local optimum.

    This method calculates and adds the following node attributes to landscape graph:

    • basin_index: The index of the local optimum reached via hill climbing from this node.
    • size_basin_greedy: For local optima nodes, this stores the total number of nodes that converge to the same local optimum as this node via hill climbing.
    • radius_basin_greedy: For local optima nodes, this stores largest the number of steps taken during the hill climb from this node to its corresponding local optimum.

    Note

    Based on the nature of greedy hill-climbing, the walk trajectory is deterministic, and the measured size of the basins is typically much smaller compared to its stochastic counterparts (e.g., adaptive walks with uniform fixation probability, a.k.a. "first-improvement" hill-climbing).

    .determine_accessible_paths() -> None

    Determines the size of basins based on accessible paths. It is automatically called with build_from_data(calculate_accessibility=True).

    This method calculates an alternative definition of basin size. For each identified local optimum, it finds all nodes in the graph from which that local optimum can be reached by any sequence of fitness-increasing steps (i.e., following the directed edges of the graph). This set of nodes constitutes the "accessible basin" or the set of ancestors. This is the type of basin analyzed in Papkou 2023.

    This method calculates and adds the following node attribute to the landscape graph:

    • size_basin_accessible: For local optima nodes, this stores the number of nodes (including the optimum itself) from which the optimum is reachable. For non-optima nodes, this value is typically 0.

    Warning

    This method can be quite computationally expensive for large landscapes (e.g., >300,000 configurations) with many local optima.

    .determine_dist_to_go(distance=None) -> None

    Calculates the distance from each node to the global optimum. It is automatically called with build_from_data(calculate_neighbor_fit=True).

    This method computes the distance between the configuration of each node and the configuration of the identified global optimum (self.go). The distance metric used depends on the landscape type (e.g., hamming_distance for boolean/sequence, mixed_distance for default) or can be specified explicitly from graphfla.distances.

    This method adds the following node attribute to the landscape graph:

    • dist_go: The calculated distance from the node's configuration to the global optimum's configuration.

    Parameters

    • distance : callable, optional
      A distance function from graphfla.distances to use, which calculates the distance between A configuration and ALL configurations in the landscape.

    .determine_neighbor_fitness() -> None

    Calculates mean neighbor fitness and the difference of this upon single mutation. It is automatically called with build_from_data(calculate_neighbor_fit=True).

    This method analyzes the fitness values in the immediate neighborhood of each node. It adds two attributes:

    • mean_neighbor_fit (node attribute): The average fitness value of all neighbors of each node.
    • delta_mean_neighbor_fit (edge attribute): For each directed edge from source node to target node(where target is fitter), this is the difference between the mean neighbor fitness of the target and the source. This can indicate if moving to a fitter neighbor also leads to a neighborhood with higher average fitness, and can be used to determine "evolvability-enhancing mutations" proposed in Wagner 2023, which is implemented in graphfla.analysis.calculate_evol_enhance()).

    Magic Methods

    Allows accessing the attributes of a specific configuration (node) using index notation.

    You can access a node by its integer index (position in the internal graph structure, typically 0 to n_configs - 1).

    Parameters

      The integer index or the string name of the configuration (node) to retrieve.

    Returns

    • dict
      A dictionary containing all attributes associated with the requested node (e.g., {'fitness': 2.5, 'is_lo': False, 'basin_index': 2, ...}).

      A summary string describing the landscape.

    Allows using the built-in len() function on the landscape object.

    Returns an iterator over the configuration indices (node indices) in the landscape.

    .__contains__(item) -> bool

    Checks if a configuration index (or name) exists in the landscape graph.

    Allows using the in operator to check for the presence of a node by its integer index or string name (if names are used).

    Parameters

    • item : int or str
      The integer index or string name to check for existence in the graph.

    Returns

    • bool
      It is True if a node with the given index or name exists, False otherwise.

    .__bool__() -> bool

    Determines the truthiness of the Landscape object.

    A Landscape object is considered True if it has been successfully built (via build_from_data or build_from_graph) and its underlying graph contains at least one node. Otherwise, it is considered False. Allows using the landscape directly in boolean contexts like if landscape:.

    Returns

    • bool
      It is True if the landscape is built and non-empty, False otherwise.
